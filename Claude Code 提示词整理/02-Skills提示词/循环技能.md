## 原版英文

```markdown
# /loop — schedule a recurring prompt

Parse the input below into `[interval] <prompt…>` and schedule it with `${CRON_CREATE_TOOL_NAME}`.

## Parsing (in priority order)

1. **Leading token**: if the first whitespace-delimited token matches `^\d+[smhd]$` (e.g. `5m`, `2h`), that's the interval; the rest is the prompt.
2. **Trailing "every" clause**: otherwise, if the input ends with `every <N><unit>` or `every <N> <unit-word>` (e.g. `every 20m`, `every 5 minutes`, `every 2 hours`), extract that as the interval and strip it from the prompt.
3. **Default**: otherwise, interval is `10m` and the entire input is the prompt.

Examples:
- `5m /babysit-prs` → interval `5m`, prompt `/babysit-prs` (rule 1)
- `check the deploy every 20m` → interval `20m`, prompt `check the deploy` (rule 2)
- `run tests every 5 minutes` → interval `5m`, prompt `run tests` (rule 2)
- `check the deploy` → interval `10m`, prompt `check the deploy` (rule 3)
- `check every PR` → interval `10m`, prompt `check every PR` (rule 3)
- `5m` → empty prompt → show usage

## Interval → cron

Supported suffixes: `s` (seconds), `m` (minutes), `h` (hours), `d` (days).

| Interval pattern | Cron expression | Notes |
|-----------------|----------------|-------|
| `Nm` where N ≤ 59 | `*/N * * * *` | every N minutes |
| `Nm` where N ≥ 60 | `0 */H * * *` | round to hours |
| `Nh` where N ≤ 23 | `0 */N * * *` | every N hours |
| `Nd` | `0 0 */N * *` | every N days at midnight |
| `Ns` | treat as `ceil(N/60)m` | cron minimum is 1 minute |

**If the interval doesn't cleanly divide its unit** (e.g. `7m` → `*/7 * * * *` gives uneven gaps), pick the nearest clean interval and tell the user.

## Action

1. Call `${CRON_CREATE_TOOL_NAME}` with:
   - `cron`: the expression from the table above
   - `prompt`: the parsed prompt verbatim
   - `recurring`: `true`
2. Briefly confirm: what's scheduled, the cron expression, the cadence, that recurring tasks auto-expire after 14 days, and that they can cancel with `${CRON_DELETE_TOOL_NAME}`.
3. **Then immediately execute the parsed prompt now** — don't wait for the first cron fire.
```

## 中文翻译

```markdown
# /loop — 安排定期提示词

将以下输入解析为 `[interval] <prompt…>`，并使用 `${CRON_CREATE_TOOL_NAME}` 安排它。

## 解析（按优先级）

1. **前导 token**：如果第一个空格分隔的 token 匹配 `^\d+[smhd]$`（如 `5m`、`2h`），则为时间间隔；其余部分是提示词。
2. **尾部 "every" 子句**：否则，如果输入以 `every <N><unit>` 或 `every <N> <unit-word>` 结尾（如 `every 20m`、`every 5 minutes`、`every 2 hours`），则将其提取为时间间隔，并从提示词中移除。
3. **默认**：否则，时间间隔为 `10m`，整个输入是提示词。

示例：
- `5m /babysit-prs` → 间隔 `5m`，提示词 `/babysit-prs`（规则 1）
- `check the deploy every 20m` → 间隔 `20m`，提示词 `check the deploy`（规则 2）
- `run tests every 5 minutes` → 间隔 `5m`，提示词 `run tests`（规则 2）
- `check the deploy` → 间隔 `10m`，提示词 `check the deploy`（规则 3）
- `check every PR` → 间隔 `10m`，提示词 `check every PR`（规则 3）
- `5m` → 空提示词 → 显示用法

## 间隔 → cron

支持的后缀：`s`（秒）、`m`（分钟）、`h`（小时）、`d`（天）。

| 间隔模式 | Cron 表达式 | 说明 |
|---------|------------|------|
| `Nm` 其中 N ≤ 59 | `*/N * * * *` | 每 N 分钟 |
| `Nm` 其中 N ≥ 60 | `0 */H * * *` | 四舍五入到小时 |
| `Nh` 其中 N ≤ 23 | `0 */N * * *` | 每 N 小时 |
| `Nd` | `0 0 */N * *` | 每天午夜 |
| `Ns` | 视为 `ceil(N/60)m` | cron 最小为 1 分钟 |

**如果间隔不能整除其单位**（如 `7m` 会导致 :56→:00 的间隔不均匀），选择最接近的整齐间隔并告知用户。

## 行动

1. 使用以下参数调用 `${CRON_CREATE_TOOL_NAME}`：
   - `cron`：上表中的表达式
   - `prompt`：解析后的提示词（原文）
   - `recurring`：`true`
2. 简要确认：安排了什么、cron 表达式、节奏、定期任务会在 14 天后自动过期，以及可以用 `${CRON_DELETE_TOOL_NAME}` 取消。
3. **然后立即执行解析后的提示词**——不要等待第一个 cron 触发。
```

## 价值解析

### 解析规则设计

```
输入: "5m /babysit-prs"
  ↓
检查第一个 token: "5m" 匹配 ^\d+[smhd]$ ✓
  ↓
结果: interval = "5m", prompt = "/babysit-prs"

输入: "check the deploy every 20m"
  ↓
第一个 token "check" 不匹配
  ↓
检查尾部 "every 20m" ✓
  ↓
结果: interval = "20m", prompt = "check the deploy"
```

### Cron 转换表的应用

| 用户输入 | Cron | 实际行为 |
|---------|------|---------|
| `5m` | `*/5 * * * *` | 每 5 分钟 |
| `1h` | `0 */1 * * *` | 每小时整点 |
| `2d` | `0 0 */2 * *` | 每两天午夜 |

### 对 AI 编程的启发

1. **多规则解析**：按优先级尝试多种模式
2. **智能转换**：将自然语言转为机器可执行格式
3. **即时执行**：不要等待定时触发，立即开始
