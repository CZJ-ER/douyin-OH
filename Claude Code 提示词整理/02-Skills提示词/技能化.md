## 原版英文

```markdown
# Skillify

You are capturing this session's repeatable process as a reusable skill.

## Your Session Context

Here is the session memory summary:
<session_memory>
{{sessionMemory}}
</session_memory>

Here are the user's messages during this session. Pay attention to how they steered the process:
<user_messages>
{{userMessages}}
</user_messages>

## Your Task

### Step 1: Analyze the Session

Before asking any questions, analyze the session to identify:
- What repeatable process was performed
- What the inputs/parameters were
- The distinct steps (in order)
- The success artifacts/criteria for each step
- Where the user corrected or steered you
- What tools and permissions were needed
- What agents were used
- What the goals and success artifacts were

### Step 2: Interview the User

**Round 1: High level confirmation**
- Suggest a name and description for the skill
- Suggest high-level goal(s) and specific success criteria

**Round 2: More details**
- Present the high-level steps as a numbered list
- Suggest arguments if the skill will need them
- Ask if the skill should run inline or forked
- Ask where the skill should be saved (repo vs personal)

**Round 3: Breaking down each step**
For each step, ask:
- What does this step produce that later steps need?
- What proves that this step succeeded?
- Should the user be asked to confirm before proceeding?
- Are any steps independent and could run in parallel?
- How should the skill be executed? (Direct, Task agent, Teammate, or [human])
- What are the hard constraints or hard preferences?

**Round 4: Final questions**
- Confirm when this skill should be invoked
- Ask for any gotchas or things to watch out for

### Step 3: Write the SKILL.md

### Step 4: Confirm and Save
```

## 中文翻译

```markdown
# Skillify

你正在将此会话的可重复过程捕获为可复用技能。

## 你的会话上下文

以下是会话记忆摘要：
<session_memory>
{{sessionMemory}}
</session_memory>

以下是用户在此会话期间的消息。注意他们如何引导过程的：
<user_messages>
{{userMessages}}
</user_messages>

## 你的任务

### 步骤 1：分析会话

在问任何问题之前，分析会话以识别：
- 执行了什么可重复的过程
- 输入/参数是什么
- 不同的步骤（按顺序）
- 每个步骤的成功产物/标准
- 用户在哪里纠正或引导了你
- 需要什么工具和权限
- 使用了什么 agent
- 目标是什么，成功产物是什么

### 步骤 2：采访用户

**第 1 轮：高级确认**
- 为技能建议名称和描述
- 建议高级目标（们）和具体成功标准

**第 2 轮：更多细节**
- 将高级步骤展示为编号列表
- 如果技能需要参数，建议参数
- 询问技能应该以内联还是分叉方式运行
- 询问技能应该保存在哪里（仓库 vs 个人）

**第 3 轮：分解每个步骤**
对于每个步骤，询问：
- 此步骤产生什么供后续步骤使用？
- 什么证明此步骤成功了？
- 在继续之前是否应该询问用户确认？
- 是否有任何步骤是独立的可以并行运行？
- 技能应该如何执行？（Direct、Task agent、Teammate 或 [human]）
- 有什么硬约束或硬偏好？

**第 4 轮：最后的问题**
- 确认何时应该调用此技能
- 询问是否有任何陷阱或需要注意的事项

### 步骤 3：编写 SKILL.md

### 步骤 4：确认并保存
```

## SKILL.md 格式说明

```markdown
---
name: {{skill-name}}
description: {{one-line description}}
allowed-tools:
  {{list of tool permission patterns}}
when_to_use: {{detailed description of when to invoke}}
argument-hint: "{{hint showing argument placeholders}}"
arguments:
  {{list of argument names}}
context: {{inline or fork}}
---

# {{技能标题}}
技能描述

## Inputs
- `$arg_name`: 此输入的描述

## Goal
清晰陈述的工作流目标。最好有明确的产物或完成标准。

## Steps

### 1. 步骤名称
在此步骤中要做什么。具体且可操作。必要时包含命令。

**Success criteria**: 始终包含！这是步骤完成可以继续的证明。可以是列表。

**Execution**: `Direct`（默认）、`Task agent`、`Teammate` 或 `[human]`

**Artifacts**: 此步骤产生的数据，后续步骤需要（如 PR 号、commit SHA）

**Human checkpoint**: 暂停并询问用户的情况（如合并、发送消息）

**Rules**: 工作的硬规则

...
```

## 价值解析

### Skillify 工作流

```
会话过程
    ↓
┌─────────────────┐
│  Step 1: 分析    │ ← 从会话提取关键信息
└────────┬────────┘
         ↓
┌─────────────────┐
│  Step 2: 采访   │ ← 4轮问答确认细节
└────────┬────────┘
         ↓
┌─────────────────┐
│  Step 3: 编写   │ ← 生成 SKILL.md
└────────┬────────┘
         ↓
┌─────────────────┐
│  Step 4: 确认   │ ← 用户审核后保存
└─────────────────┘
```

### 关键设计原则

| 原则 | 说明 |
|------|------|
| **先分析后询问** | 不要盲目问，先自己理解 |
| **渐进式采访** | 分 4 轮，从粗到细 |
| **必填成功标准** | 每个步骤必须有完成标志 |
| **执行模式区分** | Direct/Task/Teammate/human |

### 对 AI 编程的启发

1. **过程捕获**：把一次性的会话变成可复用的技能
2. **渐进确认**：不要一次问完所有问题
3. **成功标准**：每个步骤都需要明确的完成标志
