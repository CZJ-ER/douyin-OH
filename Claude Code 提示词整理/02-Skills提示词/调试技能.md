## 中文翻译

```markdown
# Debug Skill

帮助用户调试在当前 Claude Code 会话中遇到的问题。

## 调试日志刚刚启用

此会话的调试日志原本是关闭的。/debug 调用之前的内容未被捕获。

告诉用户调试日志现已激活，位于 `${debugLogPath}`，请他们重现问题，然后重新读取日志。如果无法重现，也可以用 `claude --debug` 重启，从启动时开始捕获日志。

## 会话调试日志

当前会话的调试日志位于：`${debugLogPath}`

${logInfo}

要获取更多上下文，请在完整文件中搜索 [ERROR] 和 [WARN] 行。

## 问题描述

${args || '用户没有描述具体问题。请阅读调试日志，总结错误、警告和值得注意的问题。'}

## 设置

记住设置位于：
* 用户设置 - ${getSettingsFilePathForSource('userSettings')}
* 项目设置 - ${getSettingsFilePathForSource('projectSettings')}
* 本地设置 - ${getSettingsFilePathForSource('localSettings')}

## 指令

1. 审查用户的问题描述
2. 最后 20 行显示调试文件格式。在整个文件中查找 [ERROR] 和 [WARN] 条目、堆栈跟踪和故障模式
3. 考虑启动 ${CLAUDE_CODE_GUIDE_AGENT_TYPE} 子 agent 以了解相关 Claude Code 功能
4. 用通俗易懂的语言解释你的发现
5. 提出具体的修复建议或后续步骤
```

## 价值解析

### 调试流程图

```
┌─────────────────────────────────────┐
│          用户报告问题                 │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│     启用/读取调试日志                  │
│     (如果未启用，先启用)              │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│   搜索 [ERROR] 和 [WARN]              │
│   提取堆栈跟踪和故障模式               │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│   分析问题 + 查阅相关文档              │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│   提出具体修复建议                    │
└─────────────────────────────────────┘
```

### 关键调试原则

| 原则 | 说明 |
|------|------|
| **先启用日志** | 如果日志未开启，先开启再重现问题 |
| **关键词搜索** | [ERROR] 和 [WARN] 是关键线索 |
| **堆栈跟踪** | 顺着堆栈找出问题根源 |
| **设置位置** | 了解不同层级的配置文件 |

### 对 AI 编程的启发

1. **日志先行**：调试问题前先确保有足够的日志
2. **关键词定位**：[ERROR] 和 [WARN] 是最佳起点
3. **用户配合**：让用户重现问题才能获取完整上下文
